During the Helm→ArgoCD migration on 2026-07-02, `argocd-secret` in namespace `argocd` was recreated instead of adopted. The mittwald secret-generator (`secret-generator.v1.mittwald.de/autogenerate: webhook.github.secret`) minted a new webhook secret, breaking GitHub webhook deliveries (400, HMAC verification failed) until the GitHub hook (id 605571822 on nijave/vmubtkube-a) was patched with the new value on 2026-07-03. The recreation also rotated the ArgoCD admin password and session signing key.

**Why:** any future recreation of `argocd-secret` silently breaks GitHub webhooks the same way.

**How to apply:** if webhooks 400 with "HMAC verification failed", compare `autogenerate-generated-at` on the secret vs. when GitHub last had its secret set; fix by pushing the cluster value to the GitHub hook config. Durable fix: source the webhook secret from Bitwarden via ExternalSecret (see [[serviceip-routable]] repo patterns / the SearXNG Brave API key ExternalSecret) instead of autogenerating.
